# Range Tracker — Design Guide

> Snapshot config: **Intensity blocks** + **Sky (Ghibli) palette** + all pattern aids ON + comfortable density.
> This document is written for an AI agent so it can recreate or extend the design without ambiguity.

---

## 1. Product context

**Range Tracker** is one view of *Mood tracker* — a personal app for logging emotional state every hour over a week. Each cell in the grid is a single hourly entry: a mood letter (A → D) plus a short action note.

The view's job, in priority order:
1. **Surface patterns fast.** Recurring slumps, good mornings, bad afternoons, week-over-week rhythms — visible in one glance.
2. **Drill into context after.** Once a pattern is noticed, the user wants to see *what they were doing* at those hours.

Everything below is in service of (1) first, (2) second.

---

## 2. Mood scale

Mood is a **single ordinal axis of intensity**, not a 2D valence×arousal grid.
8 levels, A → D, best → worst:

| Level | Name (vi)                       | Meaning                       |
|-------|---------------------------------|-------------------------------|
| `A`   | high energy                     | tỉnh, hứng khởi, đỉnh         |
| `B+`  | thả lỏng                        | đầu óc nhẹ, dễ chịu           |
| `B`   | bình thường, ổn định            | baseline neutral              |
| `B-`  | việc bớt, chưa mãn nguyện       | hơi tù, không vui rõ          |
| `C+`  | mệt, hiệu suất giảm             | bắt đầu xuống                 |
| `C`   | căng thẳng                      | stressed                      |
| `C-`  | kiệt sức                        | exhausted                     |
| `D`   | burnout                         | true alarm                    |

`moodIdx(level) → 0..7`. Use this for color ramps, sorting, averaging.

## 3. Layout system

Fixed structure — do not change without reason.

```
┌─ TopBar (48px) ────────────────────────────────────────────────────┐
├─ FilterBar (~70px) ───────────────────────────────────────────────┤
├─ Frame ────────────────────────────────────┬─ DetailPanel (340px) ┤
│  ┌─ corner ─┬ DayHead ┬ DayHead ┬ ... ──┐ │  (only when selected)  │
│  │          │ T2  4   │ T3  5   │ ...   │ │                        │
│  │          │ minimap │ minimap │ ...   │ │                        │
│  ├ gutter ──┼─────────┼─────────┼───────┤ │                        │
│  │ 00       │  entry  │  entry  │       │ │                        │
│  │ 01 ◦     │         │  entry  │       │ │                        │
│  │ 02       │         │         │       │ │                        │
│  │ ...      │         │         │       │ │                        │
│  │ 23       │         │         │       │ │                        │
└──┴──────────┴─────────┴─────────┴───────┴─┴───────────────────────┘
```

- **24h grid**, hour 0 at top, hour 23 at bottom. Always show all 24 rows even if some are empty.
- **Days as columns**, 7 columns (Mon → Sun). Day labels in mono small-caps + large day number.
- **Hour gutter** is a sticky column on the left (`56px`). Hours `00..23` in mono. The hourly average ribbon (a 4px-wide colored sliver) sits inside each hour cell.
- Day columns share equal width via `grid-template-columns: 56px repeat(7, 1fr)`. **Never set fixed width on day columns** — alignment relies on the grid.
- An entry is positioned absolutely inside its day column at `top = entry.h * rowH`, `height = entry.dur * rowH - 2px`. Width is `100% - 8px` (4px inset on both sides).

Row height: `rowH = 36px` (default) or `28px` (compact rows tweak).

### 4.1 `mode: "intensity"` — Intensity blocks

Each entry renders as a **solid color block** that fits within column width. The block is a flex row:

```
┌────────────────────────────────────────┐
│ [color  | B+  ] action text truncated… │   ← 1 line, ellipsis on overflow
└────────────────────────────────────────┘
   ^ 3px left border in palette FG color
   ^ background filled with palette BG color (translucent)
```

- Mood label uses the palette's **FG** color (saturated).
- Background uses the palette's **BG** color (muted/translucent on dark bg).
- Action text is **always truncated** to fit column width — `white-space: nowrap; overflow: hidden; text-overflow: ellipsis`. This is the core fix for the original problem (text overflowing column → breaking column alignment).
- Full text is available via `title` tooltip and on click → DetailPanel.

Other modes (not active):
- `heatmap` — pure color tile, no text. Strongest pattern signal.
- `compact` — translucent panel, no fill, only mood letter + truncated text.
- `bar` — width-of-fill = severity. Width grows from 12% (A) to 100% (D).

### 4.2 `palette: "sky"` — Sky (Ghibli landscape)

Inspired by a Ghibli scene: deep cobalt sky, cream clouds, meadow yellow, poppy red.

| Mood | FG (border + label)         | BG (cell fill)                              | Visual reading             |
|------|------------------------------|---------------------------------------------|----------------------------|
| `A`  | `#aacde8` pale sky           | `oklch(0.55 0.10 230 / 0.45)`               | Calm pale blue — open sky  |
| `B+` | `#7fb4dc` mid sky            | `oklch(0.50 0.12 240 / 0.55)`               | Soft blue — relaxed        |
| `B`  | `#f7f3e8` cream cloud        | `oklch(0.55 0.02 85 / 0.35)`                | Warm cream — neutral       |
| `B-` | `#e8c63a` meadow yellow      | `oklch(0.55 0.13 90 / 0.65)`                | Yellow — slight ground     |
| `C+` | `#e88a3a` orange             | `oklch(0.55 0.15 60 / 0.75)`                | Warm orange — pressure     |
| `C`  | `#d94a2a` poppy              | `oklch(0.52 0.17 35 / 0.85)`                | Vermilion — alarm rising   |
| `C-` | `#a83020` deeper poppy       | `oklch(0.45 0.17 25 / 0.90)`                | Deep red — burnout edge    |
| `D`  | `#6a1810` darkest poppy      | `oklch(0.35 0.16 22 / 0.95)`                | Wine — true burnout        |

This is a **diverging palette**: `A`/`B+` are cool (blues = restful), `B` is neutral cream, `B-`→`D` is warm (yellows → reds = activating/warning). It treats good and bad states as qualitatively different, not merely "less red" vs "more red."

Other palettes available (keep in code, switchable via tweak):
- `calm` — diverging cool blue → cream → red. Most general-purpose. Default fallback.
- `monoAlert` — A..C+ all desaturated grey-blue, only C..D go warm. Strictest signal — red is the only thing that pops.
- `classic` — original green → yellow → red sequential ramp. Kept for parity, but tends to look mucky on dark bg.

When choosing or designing a new palette: keep the **FG** saturated (≥0.13 chroma for accent rows, ≥0.04 for neutral rows), keep the **BG** lower lightness (`L 0.35–0.55`) with alpha `0.4–0.95` so it sits on the dark frame without drowning the text. Maintain monotonic chroma growth from "good" toward "bad" so severity reads through color even for users with mild color vision deficiencies.

### 4.3 Pattern aids (all ON)

Each is independent, layered on top of the grid. Designed so any subset can be toggled.

#### `showHourlyAvg: true` — Hourly avg ribbon
A 4px-wide × 60%-height vertical sliver sits at the right edge of each hour label in the gutter. Color = average mood across all 7 days at that hour, mapped through the active palette.

> Reads: "across the week, what does **this hour** usually feel like?" — diagonal scan reveals mornings vs. afternoons.

#### `showMiniSummary: true` — Day mini-summary
Below each day's number in the header, a 24-cell horizontal heatmap strip (1 cell per hour, 6px tall). Colors come from the active palette BG.

> Reads: "what was the **shape** of this day?" — one glance per column.

#### `showAlignmentGuides: true` — Alignment guides on hover
When the cursor enters an hour cell in the gutter, that row is tinted across all 7 day columns simultaneously (`var(--hov)`).

> Reads: "compare **this hour** across all days" — the original goal of the view (alignment).

#### `highlightSlump: true` — Highlight recurring slump
The single hour with the worst weekly average (≥ C+ threshold) gets a permanent faint warm tint behind it (`var(--slump)`).

> Reads: "where is the **recurring pain**?" — instant pattern callout, no scanning required.

### 4.4 `compactRows: false`
36px row height. Generous, readable. Set `true` for 28px when a user has dense logging or wants more on screen.

## 5. Typography

| Use                           | Font              | Size / weight             |
|-------------------------------|-------------------|---------------------------|
| Brand wordmark                | JetBrains Mono    | 13px / 400, +0.02em       |
| Top nav                       | Inter             | 12.5px / 400              |
| Filter labels (allcaps)       | JetBrains Mono    | 9.5px / 400, +0.06em      |
| Day label (T2, T3, …)         | JetBrains Mono    | 10px allcaps, +0.08em     |
| Day number (4, 5, …)          | Inter             | 18px / 500, tabular-nums  |
| Hour label                    | JetBrains Mono    | 10px / 400                |
| Mood letter inside cell       | JetBrains Mono    | 10.5px / 600              |
| Action text inside cell       | Inter             | 11px / 400 or 500         |
| Detail panel time             | JetBrains Mono    | 11px / 400, +0.04em       |
| Detail panel mood             | JetBrains Mono    | 32px / 600                |
| Detail body                   | Inter             | 13px / 400, line 1.5      |

Avoid serif. JetBrains Mono carries everything timestamp-y or systematic; Inter carries body and human-typed content.

## 7. Interaction contracts

- **Click an entry** → `selected = entry`, opens DetailPanel on the right (340px). Overrides selection on next click. Close via `✕`.
- **Hover entry** → 1px outline, slight lift, `box-shadow` halo. Tooltip = `mood · action`.
- **Hover hour gutter cell** (only if `showAlignmentGuides`) → that row tints across all 7 days.
- **Tweaks panel** opens via the host's edit-mode toolbar toggle. Settings persist into the `EDITMODE` JSON block on disk.

Selected entry has a 1.5px ring in `--fg`, raised z-index (3).

---

## 8. Composition rules for new features

- **Never let an entry overflow its column.** Every entry must self-bound at column width. If text is too long → truncate; full text lives in tooltip + DetailPanel.
- **Never break the 24-row grid.** Adding new info at hour-level should stack inside the row (e.g., the avg ribbon sits inside the gutter row). Adding new info at day-level should stack inside the day header (like the mini-summary).
- **New summary signals go in the header, not in the cells.** The cells are a high-density data layer; headers are an aggregation layer. Don't mix.
- **A new palette must satisfy:** monotonic ordering by mood index when desaturated, distinguishable on dark bg, FG color usable for both border (≥3px) and text label (10.5px / 600). Test with `monoAlert` reading: turn the chroma down to 0 — does the lightness alone still preserve order?
- **Tweaks default state should look complete on its own.** Toggling everything off should never leave a broken page; it should leave a calm minimal grid. Toggling everything on (current screenshot state) is the "loud" recommended view.
- **No emoji, no gradient backgrounds, no rounded-corner-with-left-accent containers, no decorative SVG illustrations** (unless explicitly part of the brand for that screen — Sky View has them, Range Tracker doesn't).

## 9. Data shape

```ts
type Entry = {
  d: number,         // dayIdx 0..6 (Mon..Sun)
  h: number,         // hour 0..23
  dur: number,       // duration in hours, default 1
  mood: 'A'|'B+'|'B'|'B-'|'C+'|'C'|'C-'|'D',
  action: string,    // free text, vi
  reason?: string,   // optional second-order note
};
```

`DAYS[d]` provides label + numeric date for headers.
`PALETTES[name]` is `{ fg: Record<mood,string>, bg: Record<mood,string>, name: string }`.

Active palette is mirrored to `window.MOOD_COLOR` and `window.MOOD_BG` on every render so any consumer (including new components added later) reads a single source of truth.
